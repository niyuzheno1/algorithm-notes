What's radial basis function?

A radial basis function (RBF) is a real-valued function that depends only on the distance from a fixed point in the input space, called the center. In machine learning and function approximation, RBFs are often used as building blocks for constructing more complex models. RBFs are frequently used in the context of kernel methods, where they are used to define the kernel function that measures the similarity between data points in the input space. One common application of RBFs is in the construction of neural networks, where they are often used in the hidden layer to provide non-linear mapping of the input data.

def rbf(x, c, s):

CART stands for "Classification and Regression Trees," which is a machine learning algorithm developed by Leo Breiman, Jerome Friedman, Richard Olshen, and Charles Stone in the 1980s. CART is a generalization of the decision tree algorithm, where each internal node of the tree represents a test on an input feature, and each leaf node represents a prediction of the target variable.

CART can be used for both classification and regression problems, and it uses a binary recursive partitioning algorithm to divide the input space into smaller regions. The algorithm selects the input feature and the threshold that produces the best split of the data, based on a specified loss function.

How do we define boosting mathematically?

Boosting is a machine learning meta-algorithm that aims to improve the accuracy of a weak learner by iteratively training an ensemble of weak learners, each focusing on the samples misclassified by the previous ones. It can be defined mathematically as follows:

Given a training dataset ${(x_1, y_1), (x_2, y_2), \ldots, (x_n, y_n)}$, where $x_i$ represents the $i$-th input and $y_i$ represents the corresponding output, and a weak learning algorithm $\mathcal{A}$ that can produce a hypothesis $h: \mathcal{X} \rightarrow \mathcal{Y}$ that is slightly better than random guessing, boosting constructs an ensemble $H(x) = \sum_{m=1}^M \alpha_m h_m(x)$ of $M$ hypotheses $h_1, h_2, \ldots, h_M$ weighted by coefficients $\alpha_1, \alpha_2, \ldots, \alpha_M$, where $M$ is the total number of boosting rounds.

The idea of boosting is to assign higher weights to misclassified samples, so that the weak learner can focus more on these samples and try to classify them correctly in the next round. Initially, all samples are assigned equal weights $w_i = \frac{1}{n}$, and in each boosting round, the weak learner is trained on the current set of weighted samples and produces a new hypothesis $h_m$ that minimizes the weighted error:

$$h_m = \operatorname*{argmin}{h \in \mathcal{H}} \sum{i=1}^n w_i \cdot \mathbb{I}[h(x_i) \neq y_i],$$

where $\mathbb{I}$ is the indicator function that returns $1$ if its argument is true, and $0$ otherwise. The error of the $m$-th hypothesis is then computed as:

$$\epsilon_m = \sum_{i=1}^n w_i \cdot \mathbb{I}[h_m(x_i) \neq y_i].$$

The coefficient $\alpha_m$ is then computed as:

$$\alpha_m = \frac{1}{2} \log \frac{1 - \epsilon_m}{\epsilon_m}.$$

The weights are then updated as:

$$w_i \leftarrow w_i \cdot \exp\left(-\alpha_m y_i h_m(x_i)\right),$$

where $y_i \in {-1, 1}$ is the label of the $i$-th sample. This updates the weights of the misclassified samples so that they have higher weight in the next round, and the weights of the correctly classified samples are decreased. The weights are then normalized so that they sum to 1, and the process is repeated for $M$ rounds.

The final boosted hypothesis is then given by $H(x) = \operatorname{sign}(\sum_{m=1}^M \alpha_m h_m(x))$, where $\operatorname{sign}$ is the sign function that returns $1$ for positive values and $-1$ for negative values.

How can we do line search?

Line search is a technique used in numerical optimization to determine the step size to move along the search direction, in order to minimize (or maximize) an objective function. The line search algorithm performs a one-dimensional search along the direction of the search in order to find the optimal step size that minimizes the objective function.

In general, a line search algorithm takes as input the current point (the starting point of the line search), the search direction, and the objective function. The algorithm then performs a search along the line in the search direction, evaluating the objective function at different points along the line, until it finds the point that minimizes the objective function. The step size that corresponds to the minimizing point is then returned as the output of the line search algorithm.

There are many different algorithms that can be used for line search, including:

  1. Golden section search: a classical algorithm that divides the search interval into two equal parts and retains the part that contains the minimum.

  2. Fibonacci search: another classical algorithm that uses the Fibonacci sequence to determine the search points along the interval.

  3. Armijo-Goldstein search: a simple line search algorithm that finds the largest step size that satisfies the Armijo condition, which is a sufficient decrease condition for the objective function.

  4. Wolfe conditions: a more sophisticated line search algorithm that uses two conditions, known as the Wolfe conditions, to ensure that the step size satisfies both a sufficient decrease condition and a curvature condition.

The choice of line search algorithm depends on the specific optimization problem and the properties of the objective function.
